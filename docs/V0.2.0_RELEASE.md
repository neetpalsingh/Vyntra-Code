# Vyntra Code v0.2.0 Release Notes

## 🎉 What's New

### ✨ React Webview UI with Tailwind CSS

The webview has been completely rebuilt with modern web technologies:

- **React 18** for component-based UI
- **Vite** for lightning-fast builds
- **Tailwind CSS** for beautiful, VS Code-integrated styling
- **TypeScript** for type safety

### 🔄 Streaming Responses

Real-time streaming chat responses for a more interactive experience:

- See AI responses as they're generated
- Reduced perceived latency
- Better user feedback with loading states

### 💬 Multi-Conversation Management

Manage multiple conversation threads:

- Create new conversations
- Switch between conversations
- Persistent conversation history
- Conversation sidebar with search

### 🎨 Enhanced UI Components

#### MessageList
- Markdown rendering with `react-markdown`
- Syntax highlighting with `prism-react-renderer`
- Code copy buttons
- User/Assistant message distinction

#### ChatInput
- Auto-resizing textarea
- Keyboard shortcuts (Enter to send, Shift+Enter for new line)
- Attach context button
- Character count and tips

#### Settings Panel
- Provider selection
- Model configuration
- Temperature and max tokens sliders
- RAG toggle
- Visual feedback on save

### 🔧 Technical Improvements

#### Webview Architecture
```
webview/
├── src/
│   ├── components/
│   │   ├── ChatView.tsx          # Main chat interface
│   │   ├── MessageList.tsx       # Message rendering
│   │   ├── MessageItem.tsx       # Individual messages
│   │   ├── ChatInput.tsx         # Input component
│   │   ├── ConversationList.tsx  # Sidebar
│   │   └── SettingsView.tsx      # Settings panel
│   ├── vscode.ts                 # VS Code API wrapper
│   └── App.tsx                   # Root component
├── vite.config.ts
├── tailwind.config.js
└── package.json
```

#### Messaging System
- Type-safe communication between extension and webview
- Event-based architecture
- State persistence with VS Code API

#### Streaming Implementation
- SSE (Server-Sent Events) support
- Chunk-by-chunk rendering
- Graceful fallback to non-streaming

## 🚀 Getting Started

### Installation

```bash
cd vyntra-code
npm install
```

### Development

Terminal 1 - Build webview and extension:
```bash
npm run dev
```

Terminal 2 - Run backend:
```bash
cd backend
uvicorn main:app --reload
```

Terminal 3 - Launch extension:
```
Press F5 in VS Code
```

### Build for Production

```bash
npm run build:all
npm run package
```

This creates a `.vsix` file you can install.

## 📖 Usage Guide

### Chat Interface

1. Click Vyntra icon in sidebar
2. Type your message
3. Press Enter to send
4. Watch response stream in real-time

### Multiple Conversations

1. Click conversation icon (top-left)
2. Click "+" to create new conversation
3. Click any conversation to switch
4. Delete with trash icon

### Settings

1. Click settings icon (top-right)
2. Configure:
   - Default provider
   - Default model
   - Temperature (0 = precise, 2 = creative)
   - Max tokens
   - RAG enabled/disabled
3. Click "Save Settings"

### Keyboard Shortcuts

- `Enter` - Send message
- `Shift + Enter` - New line
- `Ctrl/Cmd + K` - Focus chat input

## 🎨 UI Components

### VS Code Integration

Tailwind is configured to use VS Code CSS variables:

```css
/* Automatic dark/light theme support */
--vscode-foreground
--vscode-background
--vscode-input-background
--vscode-button-background
```

### Custom Scrollbar

Native VS Code-style scrollbars in chat and sidebar.

### Markdown Support

- **Bold**, *italic*, `code`
- Lists, tables, links
- GitHub-flavored markdown (via `remark-gfm`)

### Code Blocks

- Syntax highlighting for 100+ languages
- Copy button on hover
- Line numbers (optional)
- VS Code Dark theme

## 🔧 Configuration

### VS Code Settings

```json
{
  "vyntra.defaultProvider": "openai",
  "vyntra.defaultModel": "gpt-4-turbo-preview",
  "vyntra.ragEnabled": false,
  "vyntra.temperature": 0.7,
  "vyntra.maxTokens": 2000
}
```

### Environment Variables

```env
# backend/.env
PROXY_URL=http://localhost:8338
QDRANT_URL=http://localhost:6333
```

## 🐛 Known Issues

1. **Streaming on Windows**: May have slight delay due to buffering
2. **Large files**: Context truncation may cut important code
3. **Code copy**: Doesn't work in some themes (fallback: manual copy)
